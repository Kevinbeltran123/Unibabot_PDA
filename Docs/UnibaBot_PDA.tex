\begin{abstract}
Each semester, academic program offices at the University of Ibagu\'{e} must manually review over twenty Academic Development Plans (PDAs) to verify compliance with institutional guidelines, ABET student outcomes, and program-level competency mappings. This process is time-consuming, inconsistent, and prone to human error. We propose UnibaBot PDA, an intelligent agent that automates this verification by combining Retrieval-Augmented Generation (RAG) with a fine-tuned large language model (LLM). The institutional knowledge base---comprising ABET outcomes with their performance indicators, program-specific competencies, and per-course competency mappings---is encoded into a vector store so that the relevant criteria for each course can be retrieved at inference time. A pre-trained LLM, adapted through parameter-efficient fine-tuning (LoRA), is then used to compare the free-text content of each PDA section against the retrieved requirements and generate a structured compliance report. This paper presents the research problem, reviews the state of the art in RAG, LLM fine-tuning, and automated compliance checking, and defines the goals and research questions that guide the system's development.
\end{abstract}

\section{Introduction}
% ==============================================================

Each semester, the offices of the different academic programs at the University of Ibagu\'{e} face the demanding task of manually reviewing more than twenty Academic Development Plans (PDAs) to ensure that each one complies with the institutional guidelines and improvement directives issued for that period. This verification process is currently performed by hand, requiring faculty and administrative staff to read through each document in detail, cross-reference it against a set of normative criteria, and document any discrepancies found. The volume of documents involved, combined with the inherent subjectivity of human review, creates conditions where errors can go undetected, criteria may be applied inconsistently across reviewers, and the process as a whole introduces significant delays into the academic calendar. These inefficiencies represent a risk not only to institutional quality assurance but also to the timely preparation of courses each semester.

PDAs follow a semi-standardized template that includes identifiable sections such as learning outcomes, assessment methods, course schedules, and improvement actions from the previous semester. The normative framework against which PDAs are evaluated is not a single monolithic document but rather a structured set of interrelated requirements: seven ABET student outcomes (O1--O7) with specific performance indicators, three program-level specific competencies (C1--C3), twelve generic competencies, five Saber Pro components, six institutional dimensions, and a per-course mapping that specifies exactly which of these elements each course must address. For example, the course ``Agentes Inteligentes'' (code 22A14) must develop specific competencies C1 and C2, generic competencies 1c and 1h, Saber Pro component SP5, and institutional dimension D4. A compliant PDA for this course must demonstrate, in its free-text sections, that it addresses all of these mapped requirements. Both the PDAs and the normative criteria are primarily written in Spanish, which introduces a multilingual processing requirement.

To address this challenge, we propose UnibaBot PDA, an intelligent agent capable of automatically analyzing each PDA against the institutional requirements and generating a structured compliance report. The system receives as input a PDF document corresponding to a PDA and produces as output a detailed report identifying which sections do not meet the established requirements, the specific criteria violated, and the corrections that should be made. By automating this workflow, UnibaBot PDA aims to reduce the review time from days to minutes, eliminate human inconsistency in the application of criteria, and free program staff to focus on higher-level academic decision-making.

The system architecture combines two complementary techniques. First, Retrieval-Augmented Generation (RAG) \cite{lewis2020rag} is used to store the competency definitions, ABET outcome descriptions, and performance indicators in a vector database and retrieve the relevant criteria for each PDA section at inference time. While the per-course mappings themselves are structured data amenable to direct lookup, the descriptions and indicators that define what each competency or outcome actually requires are natural-language text that benefits from semantic retrieval. Second, a pre-trained LLM is adapted through parameter-efficient fine-tuning---specifically LoRA \cite{hu2022lora}---on a dataset of compliance-analysis examples to improve the model's ability to evaluate academic documents against normative criteria.

The remainder of this paper is organized as follows. Section~II reviews the state of the art across four areas: large language models, retrieval-augmented generation, parameter-efficient fine-tuning, and automated compliance checking. Section~III formulates the research goals and questions. Section~IV describes the proposed system architecture. Section~V outlines the planned evaluation methodology. Section~VI concludes with a discussion of expected contributions.

\section{State of the Art}
% ==============================================================

This section reviews the foundational work and recent advances that inform the design of UnibaBot PDA, organized into four areas: large language models, retrieval-augmented generation, parameter-efficient fine-tuning, and automated compliance verification.

% --------------------------------------------------------------
\subsection{Large Language Models}
% --------------------------------------------------------------

Modern large language models are built on the Transformer architecture, introduced by Vaswani et al. \cite{vaswani2017attention}. The key innovation of the Transformer is the self-attention mechanism, which allows the model to weigh the relevance of every word in a sequence against every other word, regardless of their distance. Unlike earlier recurrent neural networks that processed text sequentially, the Transformer processes all positions in parallel, making it both faster to train and better at capturing long-range relationships in text. This architecture has become the foundation of virtually all current language models.

Two main paradigms emerged from the Transformer. The first is the generative (decoder-only) approach, exemplified by the GPT family. Radford et al. \cite{radford2018gpt1} showed that pre-training a Transformer on large amounts of text and then fine-tuning it on specific tasks produces strong results. GPT-2 \cite{radford2019gpt2} scaled this approach and demonstrated that larger models can perform tasks without explicit fine-tuning, a capability known as zero-shot learning. The second paradigm is the encoder-based approach, exemplified by BERT \cite{devlin2019bert}, which reads text bidirectionally (both left-to-right and right-to-left simultaneously). BERT is particularly strong at text understanding tasks such as classification and information extraction---capabilities directly relevant to compliance verification, where the goal is to determine whether a document section satisfies a given criterion.

For projects with limited computational resources, the release of open-weight models has been transformative. LLaMA \cite{touvron2023llama} demonstrated that smaller, well-trained models (ranging from 7 billion to 65 billion parameters) can match or exceed the performance of much larger proprietary models. Mistral 7B \cite{jiang2023mistral} further improved efficiency through architectural optimizations. These models can run on freely available cloud GPU instances such as Google Colab, making them accessible for academic projects. For our system, models in the 3B--7B parameter range are the primary candidates, as they balance capability with the hardware constraints of our environment.

% --------------------------------------------------------------
\subsection{Retrieval-Augmented Generation}
% --------------------------------------------------------------

A fundamental limitation of language models is that their knowledge is fixed at training time. If the information a model needs was not in its training data---or if it has changed since training---the model cannot access it. Retrieval-Augmented Generation (RAG), introduced by Lewis et al. \cite{lewis2020rag}, addresses this by combining a language model with an external knowledge base. When the system receives a query, it first retrieves relevant documents or passages from the knowledge base, then passes both the query and the retrieved context to the language model for generation. This means the model always works with up-to-date, domain-specific information rather than relying solely on what it memorized during training.

The RAG paradigm has evolved since its introduction. Gao et al. \cite{gao2024ragsurvey} identify three stages: Naive RAG, which simply retrieves and reads; Advanced RAG, which adds query optimization before retrieval and re-ranking after retrieval; and Modular RAG, which uses interchangeable components that can be adapted to different domains. For our application, the key insight from their survey is that the way documents are split into chunks for storage in the vector database has a major impact on retrieval quality.

Fan et al. \cite{fan2024ragmeeting} emphasize that retrieval quality is the primary factor determining the quality of RAG outputs. When the system fails to retrieve the correct passage---even though it exists in the knowledge base---the downstream generation will inevitably be wrong. This finding is particularly relevant for our system: if the retrieval component fails to surface the correct ABET indicator or competency description for a given PDA section, the compliance assessment will be unreliable.

Jimeno Yepes et al. \cite{jimenoyepes2024chunking} investigated chunking strategies for financial report analysis, a domain with structural similarities to our own: both involve semi-structured documents with section headers, normative references, and defined formats. Their results show that chunking strategies that respect document structure (splitting at section boundaries rather than at fixed character counts) improve retrieval precision by 15--20\%. This informs our approach to encoding the ABET outcomes and competency definitions, where each outcome-indicator pair forms a natural chunk.

Sun et al. \cite{sun2025compliance} present a compliance checking framework built directly on RAG, where regulatory clauses are retrieved to augment LLM-based assessment. Their system outperforms approaches that rely solely on the language model's internal knowledge, confirming that external retrieval of normative criteria is beneficial for compliance tasks. This architecture is directly analogous to our proposed approach for PDA verification.

In our system, the knowledge base contains the textual descriptions of ABET outcomes and their indicators, competency definitions, and Saber Pro component descriptions. The per-course mappings (which competencies apply to which course) are structured data stored in JSON format and accessed through direct lookup by course code. RAG is used specifically for the semantic matching step: given a free-text PDA section, the system retrieves the most relevant competency and outcome descriptions to determine whether the section adequately addresses the required criteria.

\subsection{Parameter-Efficient Fine-Tuning}
% --------------------------------------------------------------

While pre-trained language models have broad capabilities, they often benefit from additional training on domain-specific data to improve performance on specialized tasks. However, fully retraining all the parameters of a large model requires substantial computational resources---for example, a 7-billion-parameter model needs over 80GB of GPU memory for full fine-tuning, far beyond what is available on free cloud platforms.

Parameter-efficient fine-tuning methods solve this problem by training only a small number of additional parameters while keeping the original model frozen. LoRA (Low-Rank Adaptation), proposed by Hu et al. \cite{hu2022lora}, is the most widely used approach. The core idea is that the changes needed to adapt a pre-trained model to a new task can be represented by small, low-rank matrices that are added to the existing model weights. In practice, this means training less than 1\% of the total parameters while achieving results comparable to full fine-tuning. Importantly, the adapted weights can be merged back into the original model after training, so there is no additional cost at inference time.

QLoRA, introduced by Dettmers et al. \cite{dettmers2023qlora}, extends LoRA by first compressing the base model to 4-bit precision before applying the low-rank adapters. This reduces memory requirements further---a 7B-parameter model that would normally require 16GB fits in approximately 6GB---making fine-tuning feasible on free-tier Google Colab T4 GPUs with 16GB of VRAM. Dettmers et al. showed that this aggressive compression introduces minimal quality loss, with their QLoRA-trained models achieving 99.3\% of the performance of models trained with full precision.

Instruction tuning, demonstrated by Wei et al. with FLAN \cite{wei2022flan}, showed that training a model on datasets formatted as instruction-response pairs produces models that can follow new instructions they have not seen before. This is the format we plan to use for our compliance-analysis dataset: each example consists of an instruction (``Evaluate whether this PDA section addresses competency C1 and ABET indicator 1.1'') paired with a structured response that identifies the compliance status and specific issues.

Taori et al. \cite{taori2023alpaca} demonstrated that such instruction-tuning datasets can be generated economically using a more capable model. They used GPT-3.5 to create 52,000 synthetic instruction-response pairs and fine-tuned a smaller LLaMA 7B model on this data, achieving competitive quality at a cost under \$600. This self-instruct approach \cite{wang2022selfinstruct} is applicable to our dataset construction: we can use a capable model to generate synthetic compliance-analysis examples from real PDA sections and the structured competency mappings, producing training data without expensive manual annotation.

% --------------------------------------------------------------
\subsection{Automated Compliance Checking}
% --------------------------------------------------------------

Automated compliance verification has been explored in several domains, though its application to academic document review remains largely unexplored.

Amaral Cejas et al. \cite{amaral2023gdpr} applied NLP methods to check data processing agreements against GDPR requirements. Their system extracts obligations from regulatory text, maps them to document sections, and classifies each section as compliant or non-compliant. While their domain differs from ours, their methodology---decomposing normative requirements and then classifying document sections against them---provides a useful pattern for PDA verification.

Zheng et al. \cite{zheng2025construction} address construction contract review using knowledge graph-enhanced LLMs. They build a structured representation of construction regulations and use it to augment LLM reasoning when evaluating contract clauses. Their finding---that combining structured knowledge with LLM capabilities produces more accurate assessments than either approach alone---directly supports our hybrid architecture, which combines structured course-competency mappings with LLM-based text analysis.

Savelka et al. \cite{savelka2023legal} found that LLMs can identify regulatory provisions and obligations in legal texts with high accuracy even without domain-specific training (zero-shot). This suggests that the language understanding capabilities of modern LLMs provide a strong starting point for compliance tasks, which can be further improved through fine-tuning on domain-specific examples.

Sun et al. \cite{sun2025docagent} propose DocAgent, a framework that decomposes complex document analysis into sub-tasks handled by specialized components. This modular pattern is relevant to PDA verification, where different sections (learning outcomes, assessment methods, improvement actions) may require different analysis strategies.

A key gap in the existing literature is the absence of systems designed for academic compliance verification, particularly in the context of ABET-accredited programs and Spanish-language documents. Our work addresses this gap by combining structured competency data with RAG-enhanced LLM analysis, applied specifically to the domain of university PDA review.

\section{Research Goals and Questions}
% ==============================================================

\subsection{General Objective}

To design and implement an intelligent agent that automatically verifies the compliance of Academic Development Plans (PDAs) against the institutional competency framework---including ABET student outcomes, program-specific competencies, generic competencies, Saber Pro components, and institutional dimensions---generating structured reports that identify non-compliant sections, the specific criteria violated, and the corrections required.

\subsection{Specific Objectives}

\begin{enumerate}
    \item To build a knowledge base that encodes ABET outcome descriptions, competency definitions, and per-course competency mappings, and to implement a RAG pipeline that retrieves the relevant criteria for each PDA section using multilingual embeddings.
    \item To construct a domain-specific dataset of compliance-analysis examples and fine-tune an open-weight LLM using LoRA to improve its ability to evaluate academic documents against normative criteria.
    \item To integrate the structured course-competency lookup, the RAG retrieval component, and the fine-tuned LLM into a unified agent that processes PDA documents end-to-end and produces structured compliance reports.
    \item To evaluate the system's accuracy and consistency against human expert reviews on a representative set of PDAs from the Systems Engineering program.
\end{enumerate}

\subsection{Research Questions}

\begin{enumerate}
    \item \textbf{RQ1:} To what extent can a RAG-augmented, fine-tuned LLM accurately identify non-compliant sections in PDAs compared to human expert reviewers?
    \item \textbf{RQ2:} How does the combination of structured competency lookup and RAG retrieval with domain-specific fine-tuning compare to using either technique in isolation for compliance verification accuracy?
    \item \textbf{RQ3:} What is the effect of fine-tuning on the model's ability to correctly interpret Spanish-language academic content and map it to the institutional competency framework?
\end{enumerate}

\section{Proposed System Architecture}
% ==============================================================

UnibaBot PDA operates as a pipeline with four stages, illustrated in Fig.~\ref{fig:architecture_placeholder}.

\textbf{Stage 1: Document Ingestion.} The system receives a PDA in PDF format, extracts the text, and segments it into sections based on the document's heading structure (learning outcomes, assessment methods, course schedule, improvement actions, etc.).

\textbf{Stage 2: Course Identification and Requirement Lookup.} The system identifies the course code from the PDA (e.g., 22A14 for ``Agentes Inteligentes'') and performs a direct lookup in the structured competency mapping to retrieve the list of required competencies, ABET indicators, generic competencies, Saber Pro components, and institutional dimensions for that course. This step does not require semantic search---it is a deterministic lookup against the JSON-encoded course-competency mapping.

\textbf{Stage 3: Semantic Retrieval and Compliance Analysis.} For each required competency or ABET indicator, the system uses RAG to retrieve the full textual description from the vector store (e.g., for ABET indicator 1.1: ``Identify the problems and applicable theories and concepts''). It then passes each (PDA section, requirement description) pair to the LLM, which determines whether the PDA section adequately addresses the requirement and, if not, explains what is missing or incorrect.

\textbf{Stage 4: Report Generation.} The per-requirement assessments are aggregated into a complete compliance report listing: requirements that are fully addressed, requirements that are partially or not addressed (with specific comments), and any PDA content that does not correspond to the expected format.

The environment is deterministic and fully observable: the same PDA evaluated against the same competency mappings will consistently produce the same compliance assessment. The system processes documents in batch mode, evaluating each PDA sequentially and producing the full set of compliance reports at the end of the run.

For the fine-tuning component, we plan to use LoRA \cite{hu2022lora} applied to a model in the 3B--7B parameter range (such as Llama 3.2 3B \cite{touvron2023llama}), selected for its accessibility on Google Colab instances. If memory constraints require it, QLoRA \cite{dettmers2023qlora} with 4-bit quantization will be used as a fallback. The training dataset will consist of instruction-response pairs generated from real PDA sections paired with the corresponding competency requirements, following the self-instruct methodology \cite{wang2022selfinstruct}.

\section{Evaluation Methodology}
% ==============================================================

The system will be evaluated along three dimensions.

\textbf{Correctness} measures the accuracy of compliance assessments. For each PDA section and each required competency, the system's judgment (compliant, partially compliant, or non-compliant) will be compared against human expert annotations. We will report precision, recall, and F1 score for the detection of non-compliant sections.

\textbf{Granularity} evaluates whether the system correctly identifies not only which sections fail but also the specific requirement violated and what correction is needed. The generated comments will be compared against reference comments written by human reviewers using ROUGE-L \cite{lin2004rouge} for surface-level overlap and BERTScore \cite{zhang2020bertscore} for semantic similarity.

\textbf{Consistency} measures whether the system produces the same assessment for the same PDA across multiple runs, and whether it applies criteria uniformly across different PDAs---a key advantage over human review, where inter-reviewer agreement is often low.

Additionally, a comparison study will evaluate three configurations: (a) a base LLM with structured lookup only (no RAG, no fine-tuning), (b) the base LLM with RAG retrieval (no fine-tuning), and (c) the full system with RAG and fine-tuning. This comparison directly addresses RQ2 by isolating the contribution of each component.

% ==============================================================
\section{Conclusion}
% ==============================================================

This paper has presented UnibaBot PDA, an intelligent agent for automated compliance verification of Academic Development Plans at the University of Ibagu\'{e}. The system combines structured lookup of per-course competency mappings, RAG-based retrieval of ABET outcome and competency descriptions, and a fine-tuned LLM for natural-language compliance analysis. The state of the art review has established that while RAG and fine-tuning techniques have been successfully applied to compliance checking in legal and regulatory domains, their application to academic quality assurance---particularly in the context of ABET-accredited programs and Spanish-language institutional documents---remains an open area. The proposed architecture, research questions, and evaluation methodology provide a clear path toward a system that can transform the PDA review process from a manual, multi-day effort into an automated workflow while improving consistency and accuracy.
